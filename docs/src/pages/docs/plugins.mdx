# Plugins

web-meta-scraper uses a plugin architecture. Each plugin extracts metadata from a specific source in the HTML. You combine plugins to build the scraper you need.

```typescript
import { createScraper, metaTags, openGraph, twitter, jsonLd } from 'web-meta-scraper';

const scrape = createScraper([metaTags, openGraph, twitter, jsonLd]);
```

## Meta Tags

Extracts metadata from standard HTML `<meta>` tags and other head elements.

```typescript
import { createScraper, metaTags } from 'web-meta-scraper';

const scrape = createScraper([metaTags]);
```

**Extracted fields:**

| Field | Source |
| --- | --- |
| `title` | `<title>` tag |
| `description` | `<meta name="description">` |
| `keywords` | `<meta name="keywords">` (parsed as array) |
| `author` | `<meta name="author">` |
| `favicon` | `<link rel="icon">` or `<link rel="shortcut icon">` |
| `canonicalUrl` | `<link rel="canonical">` |

## Open Graph

Extracts [Open Graph](https://ogp.me/) protocol metadata from `og:` meta tags.

```typescript
import { createScraper, openGraph } from 'web-meta-scraper';

const scrape = createScraper([openGraph]);
```

**Extracted fields:**

| Field | Source |
| --- | --- |
| `title` | `og:title` |
| `description` | `og:description` |
| `image` | `og:image` |
| `url` | `og:url` |
| `type` | `og:type` (`website`, `article`, `profile`, `book`, `music`, `video`) |
| `siteName` | `og:site_name` |
| `locale` | `og:locale` |

## Twitter Cards

Extracts [Twitter Cards](https://developer.x.com/en/docs/twitter-for-websites/cards) metadata from `twitter:` meta tags.

```typescript
import { createScraper, twitter } from 'web-meta-scraper';

const scrape = createScraper([twitter]);
```

**Extracted fields:**

| Field | Source |
| --- | --- |
| `title` | `twitter:title` |
| `description` | `twitter:description` |
| `image` | `twitter:image` |
| `card` | `twitter:card` (`summary`, `summary_large_image`, `app`, `player`) |
| `site` | `twitter:site` |
| `creator` | `twitter:creator` |

## JSON-LD

Extracts [JSON-LD](https://json-ld.org/) structured data from `<script type="application/ld+json">` tags.

```typescript
import { createScraper, jsonLd } from 'web-meta-scraper';

const scrape = createScraper([jsonLd]);
```

Parsed data is available as `structuredData` in the result. Supports Schema.org types including:

- `Article`
- `Product`
- `Organization`
- `Person`
- `WebSite`
- `FAQPage`
- `BreadcrumbList`

The plugin also handles `@graph` notation (multiple structured data items in a single script tag).

```typescript
const metadata = await scrape('https://example.com');

// structuredData is an array of JSON-LD objects
for (const item of metadata.structuredData) {
  console.log(item['@type']); // e.g. "Article", "Organization"
}
```

## Combining Plugins

Plugins run in parallel and their results are merged by priority. The order in the array does not affect priority — it's determined by the [built-in priority rules](/docs/introduction#priority-merging).

```typescript
// All of these produce the same result:
createScraper([metaTags, openGraph, twitter]);
createScraper([twitter, openGraph, metaTags]);
createScraper([openGraph, twitter, metaTags]);
```

To learn how to create your own plugins, see [Custom Plugins](/docs/custom-plugins).
